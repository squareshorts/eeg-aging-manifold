# Reproducibility Checklist

- [x] Raw EEG files are excluded from the repository and documented in `data/README.md` and `scripts/00_download_or_link_data.md`.
- [x] All fitted development parameters are saved under `results/models/ds005385_ses1_v1/`.
- [x] Frozen parameters include imputation medians, winsorization thresholds, scaler means/SDs, PCA loadings, aging-axis centroids/direction, ridge coefficients, and feature list/order.
- [x] `scripts/03_project_followup.py` and `scripts/04_external_validation.py` load frozen parameters instead of refitting transformations.
- [x] Development, follow-up, and external feature order checks are implemented in `scripts/07_sanity_checks.py`.
- [x] Metadata-like fields such as participant ID, age, sex, session, filenames, and channel counts are excluded from model feature matrices.
- [x] Session-2 and external validation data are projected only after ds005385 session-1 model artifacts exist.
- [x] External validation is labeled as a full 93-feature fixed transfer; no PCA or ridge model is refitted on ds003775.
- [x] The cohort summary table is generated by `scripts/06_make_tables.py` and saved to `tables/cohort_summary.tex` and `results/tables/cohort_summary.csv`.
- [x] The development/external performance table is generated by `scripts/06_make_tables.py` and saved to `tables/development_external_performance.tex` and `results/tables/development_external_performance.csv`.
- [x] Sanity-check results are saved to `results/tables/reproducibility_sanity_checks.csv`.

Known limitation: the manuscript still depends on the previously generated ds005385-derived feature tables in `outputs/` unless users download raw ds005385 and rerun `scripts/01_extract_features.py`.
